## 二、技术栈确定与调整

当前技术栈：

- 前端：React + Vite + TypeScript
- 后端：Python 3.13+ + FastAPI
- 状态管理：Zustand
- 图表/可视化：ECharts / Mermaid / Markdown 渲染
- 智能体与大模型：后端封装 LLM Client，当前可用 DeepSeek 测试
- 知识库：本地 Markdown + JSON 课程目录
- 测试：轻量 Python 脚本 + 前端 TypeScript build

重要调整：

- 原计划前端可用 Vue，但因组员前端初版实际使用 React，项目已经迁移为 React。
- 后端仍使用 Python + FastAPI。
- 大模型没有写死在业务逻辑中，而是通过统一 LLM Client 封装，后续可替换为 DeepSeek、讯飞星火、通义千问、OpenAI 兼容接口或其他模型。
- 多模态模型后续应作为资源生成智能体的独立工具接入，不混入普通文本 LLM Client。

## 四、智能体能力当前进展

当前已具备雏形的智能体/模块：

### 1. 意图识别智能体

作用：

- 判断用户是在闲聊、补充画像、请求生成方案、问日期、请求解释、请求资源、学习反馈等。

已优化的问题：

- `你好` 不会触发学习方案生成。
- `今天是几号` 不会写进学习画像。
- `你啥意思` 会进入澄清解释，而不是固定欢迎词。
- `我是软件工程大二学生，数据结构基础一般，想学习数据结构，为了考试通过` 会进入画像更新，而不是闲聊。

### 2. 学习画像构建模块

作用：

- 从自然语言中抽取画像字段。

当前支持的画像字段：

- 身份/专业背景
- 目标课程/知识方向
- 已有基础
- 薄弱点
- 学习目标
- 时间安排
- 学习偏好

已优化的问题：

- 不再把“我是男生”等低价值背景直接当成完整学习画像。
- 能区分“数据结构还可以”和“不会 Python”。
- 能从一句话中拆出基础、薄弱点、目标课程等信息。
- 信息不足时会追问，不会立即乱生成。

### 3. 多智能体编排雏形

当前已有 AgentOrchestrator，串联：

- ProfileAgent
- KnowledgeAgent
- DiagnosisAgent
- PlannerAgent
- ResourceAgent
- ReviewAgent

当前还是第一阶段雏形，部分结果基于 mock 数据和规则化生成。后续需要逐步接入真实 LLM 和课程知识库检索。

### 4. 学习路径规划智能体雏形

已能生成学习路径页面可展示的数据结构。

当前不足：

- 路径内容仍偏 mock。
- 后续需要根据课程知识库和学习画像生成更真实的阶段计划。

### 5. 资源生成智能体雏形

已能展示资源库，包括：

- 课程讲义
- 思维导图
- 练习题
- 拓展阅读
- 实操案例
- 视频脚本类资源雏形

当前不足：

- 内容质量仍需进一步真实化。
- 后续需要基于课程章节和 LLM 生成个性化资源。

### 6. 学习行为追踪/效果评估雏形

已新增学习追踪服务：

- 记录资源浏览
- 记录资源完成
- 记录练习结果
- 记录路径节点进度
- 记录用户反馈
- 统计学习时长
- 统计练习正确率
- 识别薄弱知识点
- 生成简单调整建议

相关接口：

```text
POST /api/feedback/event
POST /api/feedback
GET /api/learning-analytics?sessionId=xxx
```

相关文件：

- `backend/app/services/learning_tracker.py`
- `backend/tests/learning_tracker_test.py`
- `docs/architecture/learning-analytics-tracking.md`
