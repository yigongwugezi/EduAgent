# 数据结构与算法复杂度基础

## 学习目标

- 理解数据结构解决的是“如何组织数据”的问题。
- 理解算法复杂度用于衡量算法随输入规模增长的变化趋势。
- 能区分常见复杂度：O(1)、O(log n)、O(n)、O(n log n)、O(n^2)。

## 核心知识点

- 数据结构：线性表、栈、队列、树、图、散列表。
- 算法操作：增、删、查、改、遍历、排序、查找。
- 时间复杂度：关注执行次数随输入规模 n 的增长。
- 空间复杂度：关注额外内存占用。

## 易错点

- 把实际运行时间和时间复杂度混为一谈。
- 忽略循环嵌套对复杂度的影响。
- 只背复杂度，不理解数据规模变大后的增长差异。

## 练习建议

给出一段含单层循环、双层循环、二分查找的伪代码，判断时间复杂度并说明原因。
